# Руководство по стилистике иконок (Iconography Guidelines) | BITFORGE

Данный документ описывает единую визуальную систему иконок для проекта BITFORGE. Стилистика базируется на текущем логотипе, общей концепции дизайна (Dark Mode SaaS, Brutalism, Tech) и архитектуре интерфейса.

---

## 1. Базовая философия (Core Philosophy)

Иконки BITFORGE должны транслировать **инженерную точность, технологичность и минимализм**. 
Они не должны быть слишком "игривыми" или мультяшными. Мы используем строгий **Linear Style** (линейный стиль), который идеально сочетается с моноширинными шрифтами и обилием Negative Space.

*   **Метафора:** Чертеж, микросхема, неоновая вывеска, ковка (Forge).
*   **Библиотека по умолчанию:** `lucide-react` (с кастомными параметрами).

---

## 2. Технические параметры (Technical Specs)

Для сохранения консистентности все иконки должны подчиняться строгим правилам геометрии:

*   **Базовая сетка (Grid):** 24x24 px.
*   **Безопасная зона (Safe Zone):** 2 px (иконка вписывается в 20x20 px).
*   **Толщина линий (Stroke Width):** 
    *   **Основная:** `1.5px` (Придает иконкам более острый, премиальный и "инженерный" вид по сравнению со стандартными 2px).
    *   **Для мелких интерфейсных элементов (менее 16px):** `2px` (для сохранения читаемости).
*   **Скругления (Border Radius / Stroke Linejoin):** 
    *   Острые или минимально скругленные углы (`rx="1"`, `stroke-linejoin="miter"` или `round` с минимальным радиусом). Это подчеркивает брутальность и связь с "битами" и "ковкой".
*   **Заливка (Fill):** Строго `none` (прозрачная). Мы используем только контурные (outline) иконки. Заливка допускается только для критических алертов или активных стейтов (например, иконка "Play" в видео).

---

## 3. Цветовая палитра (Color System)

Иконки используют переменные Tailwind, заложенные в дизайн-систему проекта:

| Состояние | Цвет (Tailwind) | Hex / RGBA | Описание |
| :--- | :--- | :--- | :--- |
| **Default** | `text-beige/70` | `rgba(245, 245, 220, 0.7)` | Базовый цвет для большинства иконок в спокойном состоянии. |
| **Hover / Active** | `text-lava` | `#ff4d00` | Активное состояние, наведение курсора, открытые модалки. |
| **Disabled** | `text-beige/20` | `rgba(245, 245, 220, 0.2)` | Неактивные элементы, фоновые декоративные иконки. |
| **Accent Glow** | `drop-shadow-[0_0_8px_#ff4d00]` | - | Для иконок, требующих максимального внимания (CTA). |

---

## 4. Визуальная тяжесть (Visual Weight)

Визуальная тяжесть всех иконок в одном ряду должна быть одинаковой. 
*   **Баланс:** Квадратные иконки визуально кажутся больше круглых. Если рядом стоят иконка "Сервер" (квадратная) и "Глобус" (круглая), круглая должна занимать чуть больше места в сетке (например, 22px против 20px у квадратной).
*   **Детализация:** Избегайте иконок с избыточным количеством внутренних линий. Если при уменьшении до 16px линии сливаются в "кашу", иконку нужно упростить.

## 5. Примеры использования (Usage Examples)

### Навигация и UI-контролы (UI Controls)
Используются для управления интерфейсом (бургер, крестик, стрелки).
*   **Размер:** `size={24}` (для мобилок) или `size={16}` (для десктопных кнопок).
*   **Пример кода:**
    ```tsx
    import { Menu, X, ChevronDown, ArrowRight } from 'lucide-react';

    // Бургер-меню
    <Menu size={24} strokeWidth={1.5} className="text-dark" />
    
    // Стрелка в кнопке
    <ArrowRight size={16} strokeWidth={1.5} className="text-lava group-hover:translate-x-1 transition-transform" />
    ```

### Иконки услуг / Фичей (Feature Icons)
Используются в карточках услуг или в секции преимуществ.
*   **Размер:** `size={32}` или `size={40}`.
*   **Толщина:** `strokeWidth={1}` (при увеличении размера линии должны становиться тоньше, чтобы сохранять изящность).
*   **Пример кода:**
    ```tsx
    import { Code2, Database, Smartphone, Cpu } from 'lucide-react';

    // Иконка в карточке услуги
    <div className="p-4 bg-white/5 border border-beige/10 rounded-xl">
      <Cpu size={32} strokeWidth={1} className="text-lava mb-4" />
      <h3 className="text-beige font-bold">AI Integration</h3>
    </div>
    ```

### Информационные иконки (Micro-UI)
Используются рядом с текстом (телефон, email, теги).
*   **Размер:** `size={12}` или `size={14}`.
*   **Толщина:** `strokeWidth={2}` (для читаемости).
*   **Пример кода:**
    ```tsx
    import { Mail, Phone } from 'lucide-react';

    <a href="mailto:..." className="flex items-center gap-2 text-beige/70 text-xs font-mono">
      <Mail size={14} strokeWidth={2} /> Mail@bitforge.digital
    </a>
    ```

## 6. Анимация и интерактивность

Иконки не должны быть статичными. Они должны реагировать на действия пользователя:

1.  **Трансформации (Transform):**
    *   Стрелки (`ArrowRight`) должны смещаться по оси X при наведении: `group-hover:translate-x-1`.
    *   Шевроны (`ChevronDown`) должны вращаться при открытии дропдаунов: `rotate-180`.
    *   Плюсы (`Plus`) превращаются в крестики при открытии аккордеона (FAQ): `rotate-45`.

2.  **Переходы (Transitions):**
    *   Всегда используйте плавные переходы для цвета и трансформаций: `transition-all duration-300 ease-in-out`.

3.  **Кинетика (Motion):**
    *   Для крупных декоративных иконок можно использовать `motion.svg` из `motion/react` для отрисовки контура при скролле (эффект `pathLength: 1`).
